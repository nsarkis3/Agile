CSE 212 Team 3 Web Usability Form
  1. It is clear what page the user is on. On modern sites, most pages or windows include a title or other header at the top, to clearly explain what is being displayed and to reaffirm the user's location.
  2. Anything that doesn't load instantly includes a loading icon or progress bar. On modern sites, most pages load quickly, and any pages that take time to load include some indicator of progress, so the user doesn't assume that the page is broken or frozen.
  3. Website uses concise language that avoids jargon. Modern sites should be accessible to anyone, using simple language whenever possible so it is readable and easy to understand.
  4. Any strictly-necessary technical language includes an explanation or definition. Related to the previous point, technical language is sometimes necessary, but it should be defined somewhere on the site to avoid confusion.
  5. Ability to customize the experience (different font, light mode vs. dark mode, etc.). Most modern sites or applications include an option for different fonts or color modes, and this is an important accessibility consideration for people with limited vision, colorblindness, or dyslexia.
  6. Ability to go back to the previous page. Some pages spam the history with multiple instances of the same page, preventing users from pressing the back button, so it is important that modern sites retain the functionality of this button.
  7. Each site has a consistent header and footer. Consistent headers and footers help user navigate the site more effectively and predictably, so the user doesn't need to return to previous pages unnecessarily and waste time.
  8. Website has standard element (hamburger menu, dropdown, …) to indicate more options. This helps make the site more streamlined and modern, as non-standard elements like selection maps create unnecessary confusion for users.
  9. When filling forms, forms containing fields only allow for valid values (for example like dates do not allow for alphabetical values and require the format MM/DD/YY). Modern sites should be error resistant and prevent the user from entering illegal inputs that could break the site before they could be submitted.
  10. Any inaccessible sites should be grayed out in the menu. The site should provide clear indication that a link cannot be clicked, and it should prevent the user from entering links that could break the page.
  11. Website has intuitive icons (for example, a lightbulb for tips). Most modern sites use many of the same icons, so that the user is able to recognize these icons and their functions from prior knowledge, rather than needing to memorize which features perform which action for an individual site.
  12. Using the website is intuitive, so that first-time users already know what to do. If a site is not intuitive, users will need to learn and memorize the sequence of steps they need to take and how to take them, rather than simply performing the task as expected.
  13. Flexibility and efficiency of use - Frequent tasks can be done in multiple ways, such as with right clicking, keyboard shortcuts, or a button on the screen. This makes each option visible to new users, but experienced users can perform actions more efficiently using shortcuts which are commonly established across sites (such as Ctrl+p for print).
  14. Flexibility and efficiency of use - Users can set default values for frequently-inputted information, or an autofill feature is applied. When users input the same form repeatedly, they shouldn't need to manually input the same information over and over.
  15. The website uses high contrast color for background. This increases the readability of the website for the user, especially users with vision impairment or colorblindness, and this is often a requirement for modern sites.
  16. The website uses standard, legible fonts. Similar to the point above, a readable font ensures that the user will be able to read the provided text, and the font will not distract from the information.
  17. When input on a form is invalid, there is a clear indication of which input must be fixed (for example red text or outline indicating which section has errors). This is a common feature of modern sites, helping the user understand what exactly it is that they did wrong, so they don't have to check over every section and resubmit using trial-and-error.
  18. Any error should include an explanation of what the problem is and how to proceed. As with the previous explanation, this shows the user what invalid action they made and how to avoid doing it again, and if necessary, the user can search for more information on the specific error.
  19. There is a section for help in the main menu. On most sites, users try to figure out their own issues before asking for help, so they should have access to a list of helpful resources.
  20. There is a persistent element for help chat. Modern sites frequently used by inexperienced users should have a visible window allowing the user is able to solve their issues with an experienced agent, rather than needing to send an email and wait for a response.
